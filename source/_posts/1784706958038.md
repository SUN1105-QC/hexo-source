◆功能追加原则：三部分组成
1，个人想法记录：
	首先我自己将想要的功能描述出来，单独记录在一个文本文件当中。
2，AI想法整理：
	将我的个人想法文本内容，发送给AI进行进一步分析和整理，使其从个人想法文件变为功能描述文件。
3，AI指令作成：
	根据功能描述文件，作成AI指令文件。让AI工具台，可以根据这份AI指令文件，执行编码任务。


◆◆第一部分：个人想法记录
人生时间轴（Life Timeline）
一条能够回顾人生的时间长河。

我的想法是这样的，
1，大标题：默认使用 当前的时间 + 天气 为标题，包括年月日，星期，天气等。
2，内容：内容包括两个部分，文字内容和图片内容。比如输入一段文字内容：我现在在日本东京车站的咖啡厅里喝咖啡，再拍一张照片上传，用户点击保存之后，系统会自动识别，保存的时间和地点，作为整段内容的小标题，最终可以在时间轴页面中，查看所有内容，我的时间轴页面设计是，最中间有一根柱子，每一个内容都是一个时间节点，如果是单数时间，比如1：00到1：59分记录的内容，则显示在柱子左侧并且与柱子相连。如果是双数时间，比如2：00到2：59分记录的内容，则显示在柱子右侧并且与柱子相连。在时间轴显示的内容默认都只显示简略内容，比如上面提到的小标题内容即可。然后可以点击，点击之后则会显示全部内容。也可以点击全部展开按钮，则会在时间轴中，显示每条内容的全部内容，包括照片。这样我觉得会有一种记录的成就感，有一种我走过的路的感觉。
3，可以记录的场景：任何场景，比如日常出门上学，上班，比如休息日在家的一天，比如旅行等等

# 生活记录模块 - 人生时间轴（Life Timeline）

## 产品愿景

### 核心理念

> 记录每一个瞬间，连接成为人生轨迹。

与传统日记不同，人生时间轴不仅记录内容，更记录用户走过的路、看过的风景、经历过的人生故事。

其核心价值在于：

```text
照片 = 记录瞬间
文字 = 记录感受
时间 = 记录成长
地点 = 记录足迹
时间轴 = 记录人生
```

---

# 二、记录页面设计

## 1. 顶部标题自动生成

系统默认根据当前时间、天气和位置自动生成标题。

### 展示示例

```text
2026年7月22日 星期三
☀️ 晴天 31℃
东京 · 丸之内
```

或：

```text
2026/07/22（水）
☀️ 晴
日本东京站
```

### 自动获取信息

- 日期
- 星期
- 天气
- 温度
- GPS位置
- 城市信息

用户无需手动填写。

实现一种“这一刻已被正式记录”的仪式感。

---

## 2. 内容输入区域

### 文字记录

示例：

```text
今天下午在东京站旁边的咖啡厅喝咖啡。

窗外可以看到来来往往的人群。

最近工作有些忙，但坐下喝咖啡的时候，
忽然感觉时间变慢了。
```

### 图片记录

支持：

- 拍照上传
- 图片上传
- 多图上传
- 视频上传（后续扩展）

建议：

```text
最多支持9张照片
```

提供类似朋友圈的使用体验。

---

# 三、智能生成时间轴标题

这是整个功能中最具价值的设计。

## 自动识别内容

保存后系统自动分析：

- 时间
- 地点
- 关键词
- 场景

并自动生成时间轴展示标题。

### 示例一

输入：

```text
我在东京站咖啡厅喝咖啡
```

系统生成：

```text
15:26
东京站 · 咖啡时光
```

### 示例二

输入：

```text
今天第一次来到富士山
```

系统生成：

```text
09:15
山梨县 · 初见富士山
```

### 示例三

输入：

```text
和朋友一起聚餐
```

系统生成：

```text
19:40
池袋 · 好友聚餐
```

### 价值

避免时间轴变成：

```text
2026/7/22记录
2026/7/23记录
2026/7/24记录
```

而是变成：

```text
东京站 · 咖啡时光
池袋 · 好友聚餐
富士山 · 初见富士
```

让时间轴具备故事感。

# 四、人生时间轴页面设计

## 中央时间轴布局

```text
        │

左侧     │     右侧

内容1    │

          │     内容2

内容3    │

          │     内容4

内容5    │

        │
```

时间由上向下流动，形成“人生向前”的视觉表达。

---

## 奇偶时间分布规则

### 奇数小时记录

```text
1点
3点
5点
7点
```

显示在左侧。

### 偶数小时记录

```text
2点
4点
6点
8点
```

显示在右侧。

### 展示效果

```text
09:35 东京站咖啡厅
────────●
         │

         │●────────
          10:12 银座散步

11:45 公园休息
────────●

         │●────────
          12:30 午餐
```

增加时间轴的节奏感与视觉层次。

# 五、默认展示模式

时间轴默认仅展示摘要信息。

```text
● 东京站 · 咖啡时光
● 银座 · 下午散步
● 池袋 · 晚餐
● 家中 · 睡前阅读
```

特点：

- 浏览速度快
- 信息密度高
- 形成“人生目录”效果

# 六、展开模式

## 单条展开

点击某个节点后显示完整内容。

```text
东京站 · 咖啡时光
2026-07-22 15:26

📍 东京站

今天在咖啡厅坐了一下午。

最近工作有些忙，
难得有机会静下来。

[照片]
```

---

## 全部展开

右上角提供：

```text
[全部展开]
[全部折叠]
```

展开后显示：

- 全部文字内容
- 所有照片
- 所有记录详情

形成完整的人生回忆墙。

# 八、足迹地图（Life Map）

除时间轴外，增加地图模式。

## 功能说明

自动根据记录生成地图足迹。

示例：

```text
东京站
浅草寺
镰仓
大阪
京都
奈良
```

全部以地图标记方式展示。

### 用户价值

用户可以直观查看：

```text
原来我已经去过这么多地方。
```

显著提高长期使用意愿。

# 九、年度回顾功能

系统自动生成年度总结。

## 示例

```text
2026年度回顾
```

统计内容：

```text
记录次数：386次
拍摄照片：2142张
去过城市：28个
新增地点：57个
最常出现地点：东京站
记录最多月份：7月
最活跃时间：21:00
```

### AI年度总结

```text
2026年，

你在东京留下了许多足迹。

这一年共记录386个瞬间，
走过28座城市，
拍摄2142张照片。

最难忘的夏天，
是在镰仓海边吹风的那个傍晚。
```

增强用户成就感与情绪价值。

# 十一、产品核心价值

大多数产品解决效率问题。

人生时间轴解决的是：

```text
我这一年究竟经历了什么？

我这些年去过哪里？

我曾经是什么样的人？

我一路走到了哪里？
```

当用户持续记录三年、五年、甚至十年后，时间轴将不再只是数据。

它会成为一条真正属于用户自己的：

# 人生长河（Life Timeline）

记录成长、回忆与足迹。

这是该产品最具长期价值和用户粘性的核心竞争力。
