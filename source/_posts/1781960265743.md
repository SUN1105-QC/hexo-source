# GitHub 用户名修改后，GitHub Pages 访问路径修改流程

## 1. 修改 GitHub 用户名

先在 GitHub 账号设置中修改用户名。

例如：

```text
旧用户名：Sun1105
新用户名：SUN1105-QC
```

修改后，GitHub Pages 的主站访问地址也会随之变化。

```text
旧地址：https://sun1105.github.io
新地址：https://sun1105-qc.github.io
```

---

## 2. 修改 `XXX.github.io` 仓库名

GitHub Pages 主站仓库名必须和用户名保持一致。

格式：

```text
用户名.github.io
```

例如：

```text
SUN1105-QC.github.io
```

对应访问地址：

```text
https://sun1105-qc.github.io
```

注意：

```text
XXX 必须与 GitHub 用户名相同
```

否则 GitHub 会把它识别为项目页面，访问路径会变成：

```text
https://用户名.github.io/仓库名/
```

---

## 3. 修改项目目录下的 `_config.yml`

修改 Hexo 项目的部署仓库地址。

```yaml
deploy:
  type: git
  repo:
    # ① GitHub Pages 仓库：主站
    github: git@github.com:SUN1105-QC/SUN1105-QC.github.io.git,master

    # ② Vercel 部署仓库：用于 Vercel 自动构建
    vercel: git@github.com:SUN1105-QC/hexo-deploy.git,main
```

如果是 GitHub Pages 主站，通常还需要确认：

```yaml
url: https://sun1105-qc.github.io
root: /
```

---

## 4. 修改项目目录下的 `deploy.yml`

同步修改 GitHub Actions 中的部署目标仓库。

```yaml
jobs:
  build:
    runs-on: ubuntu-latest
    strategy:
      matrix:
        repo:
          - { name: "SUN1105-QC/SUN1105-QC.github.io", branch: "master" }  # GitHub Pages 仓库
          - { name: "SUN1105-QC/hexo-deploy", branch: "main" }             # 备份 / CI 仓库
```

---

## 5. 重新生成并部署

修改完成后，在项目根目录执行：

```bash
hexo clean
hexo generate
hexo deploy
```

简写：

```bash
hexo cl
hexo g
hexo d
```

---

## 6. 最后检查 GitHub Pages 设置

进入 GitHub Pages 仓库：

```text
Settings
↓
Pages
```

确认：

```text
Branch：master 或 gh-pages
Folder：/ root
```

这里的分支必须和 `_config.yml`、`deploy.yml` 中设置的分支一致。

---

## 核心注意点

```text
GitHub 用户名
=
XXX.github.io 仓库名中的 XXX
=
_config.yml 中的远程仓库路径
=
deploy.yml 中的部署仓库路径
```

只要这几处保持一致，GitHub Pages 访问地址就可以正常恢复。
