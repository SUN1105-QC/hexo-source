# 1. 部署总览

## 1.1 手册目标

本手册用于指导从零开始完成一个典型 Web 项目的部署上线工作，目标包括：

- 购买并准备部署所需账号与资源
- 配置服务器基础环境
- 部署数据库、后端服务、前端资源
- 配置 Nginx、域名与 HTTPS
- 建立基础运维能力，如日志、备份、监控与自动恢复

---

## 1.2 总览目录

1. 准备账号与基础资源  
2. 购买域名并完成基础解析准备  
3. 创建并初始化 Lightsail 服务器  
4. 部署 MySQL 8 数据库  
5. 部署后端服务  
6. 部署前端并配置 Nginx  
7. 配置域名与 HTTPS  
8. 配置日志、备份、监控与运维  
9. 验证上线结果与后续维护  

---

## 1.3 整体流程图

```text
准备账号
   ↓
购买域名
   ↓
购买并配置服务器
   ↓
安装基础环境
   ↓
部署数据库
   ↓
部署后端服务
   ↓
部署前端资源
   ↓
配置 Nginx 反向代理
   ↓
域名解析到服务器
   ↓
申请 SSL 证书并启用 HTTPS
   ↓
配置备份、监控、日志
   ↓
上线验证与长期维护
```

# 2. 准备账号与基础资源

## 2.1 本章目标

完成部署前所需平台账号和基础资源准备，为后续购买域名、创建服务器和代码部署打下基础。

---

## 2.2 需要准备的账号

通常需要以下账号：

- AWS 账号
- 域名服务商账号
- GitHub / GitLab 代码仓库账号
- 常用邮箱
- 可用于支付的银行卡或信用卡
- SSH 工具（如 Windows Terminal、PuTTY、MobaXterm、iTerm2）

---

## 2.3 AWS 账号申请

### 平台地址

- <https://aws.amazon.com/>

### 操作步骤

1. 打开 AWS 官网  
2. 点击 **Create an AWS Account**  
3. 输入邮箱、账号名称、密码  
4. 完成邮箱验证  
5. 填写联系人信息  
6. 填写支付方式  
7. 完成手机验证  
8. 选择基础支持计划  
9. 登录 AWS 控制台  

### 建议项

- 使用公司邮箱注册
- 开启 MFA 多因素认证
- 根账号仅用于重要管理操作
- 日常运维建议创建 IAM 子账号

### 初始设置建议

- 配置 IAM 用户
- 设置账单提醒
- 开启 MFA
- 记录账号信息并安全保管

---

## 2.4 域名服务商账号申请

### 可选平台

- Namecheap
- GoDaddy
- 阿里云万网
- 腾讯云域名
- Cloudflare Registrar

### 操作步骤

1. 打开域名服务商官网
2. 点击注册
3. 使用邮箱完成注册
4. 完成邮箱验证或手机验证
5. 进入用户后台
6. 完成实名认证（如平台要求）

---

## 2.5 代码仓库账号准备

### 可选平台

- GitHub
- GitLab
- Gitee

### 操作建议

- 确保项目代码已推送到仓库
- 准备好部署分支
- 如为私有仓库，准备 SSH Key 或 Access Token
- 为服务器部署用户配置仓库访问权限

---

## 2.6 建议的初始安全设置

- 所有重要平台开启 MFA
- 不共用主账号密码
- 将支付相关账号与技术操作账号分离
- 将服务器登录密钥妥善保管
- 不将敏感信息直接写入公开代码仓库

---

## 2.7 本章验证方式

- 能成功登录 AWS 控制台
- 能成功登录域名管理后台
- 能成功登录代码仓库平台
- 可以访问项目仓库
- 已准备好支付和邮箱验证能力

---

## 2.8 生活化理解

这一章像是：

> **先把开店需要的各种证件、平台后台账号和管理权限办齐。**

你还没有开始真正装修店面，但至少要先有：

- 商场管理后台账号
- 招牌购买渠道
- 仓库资料管理入口
- 员工手册存放位置

# 3. 购买域名并完成初始设置

## 3.1 本章目标

购买项目域名，并完成基础账户验证与 DNS 管理准备。

---

## 3.2 域名购买平台

推荐平台：

- Namecheap
- GoDaddy
- 阿里云
- 腾讯云
- Cloudflare Registrar

---

## 3.3 域名购买步骤

1. 登录域名服务商官网
2. 搜索想要的域名
3. 选择可注册的域名后缀
4. 将域名加入购物车
5. 选择购买年限
6. 关闭不必要的增值服务
7. 完成支付
8. 进入域名管理后台

---

## 3.4 推荐选择项

- 优先选择 `.com`
- 企业项目可考虑 `.com` / `.cn`
- 自动续费建议开启
- Whois 隐私保护按需开启
- 域名名称尽量简短、易读、易拼写

---

## 3.5 域名初始设置

购买完成后，建议立即完成以下初始化操作：

- 完成邮箱验证
- 完成实名认证（如有要求）
- 找到 DNS 管理入口
- 准备后续添加 A 记录
- 记录域名到期时间与续费周期

---

## 3.6 本章验证方式

- 域名已成功购买
- 域名后台可正常访问
- DNS 解析管理入口已找到
- 域名状态正常可用

---

## 3.7 生活化理解

这一章像是：

> **先给店铺买下正式招牌名称。**

顾客记住的是招牌名字，不是门牌号。  
域名就是用户记住并访问你项目的那个“店名”。

# 4. 创建并初始化 Lightsail 服务器

## 4.1 本章目标

创建 Ubuntu 服务器实例并完成最基础的运行环境初始化。

---

## 4.2 购买与创建实例

### 平台地址

- <https://lightsail.aws.amazon.com/>

### 操作步骤

1. 登录 AWS 控制台
2. 进入 Lightsail
3. 点击 **Create instance**
4. 选择 Region
5. 选择 Platform：**Linux/Unix**
6. 选择 Blueprint：**Ubuntu 22.04 LTS**
7. 选择实例套餐
8. 设置实例名称
9. 点击创建实例

### 推荐选项

- 系统：Ubuntu LTS
- 区域：靠近主要用户群
- 套餐：初期按最低可用配置起步，后续可升级

---

## 4.3 绑定静态 IP

### 操作目标

为服务器绑定一个固定不变的公网 IP。

### 操作步骤

1. 进入 Lightsail 实例详情页
2. 找到 Networking 或 Static IP 页面
3. 创建一个新的 Static IP
4. 将该静态 IP 绑定到目标实例

### 作用

- 避免服务器 IP 改变
- 便于域名解析绑定
- 便于长期稳定访问

---

## 4.4 配置防火墙

### 建议开放端口

- 22：SSH
- 80：HTTP
- 443：HTTPS

### 原则

- 只开放必要端口
- 数据库端口通常不对公网开放
- 非必要服务不要暴露外网

---

## 4.5 SSH 登录

### 操作步骤

1. 在 Lightsail 后台获取默认 SSH 登录方式
2. 下载或准备密钥文件
3. 使用终端工具连接服务器
4. 完成首次登录测试

---

## 4.6 更新系统软件

### 常用命令

```bash
sudo apt update && sudo apt upgrade -y
```

### 作用

- 获取最新安全补丁
- 修复系统已知问题
- 同步软件包版本

---

## 4.7 创建普通部署用户

### 原则

不建议长期直接使用 root 进行部署操作。

### 建议做法

- 创建普通部署用户
- 为该用户授予 sudo 权限
- 使用普通用户完成日常部署和维护

---

## 4.8 安装基础工具

常见基础工具包括：

- git
- curl
- wget
- unzip
- vim / nano
- htop
- net-tools

---

## 4.9 推荐初始化内容

常见初始化项目：

- `apt update && apt upgrade`
- 安装 nginx
- 安装运行时环境
  - Node.js / Java / Python
- 安装 git
- 安装 mysql-server 或准备 Docker
- 配置时区
- 配置 Swap
- 配置日志轮转

---

## 4.10 本章验证方式

- SSH 可正常登录
- 静态 IP 已绑定成功
- 系统可正常更新
- 普通部署用户可正常使用
- 基础工具安装完成

---

## 4.11 生活化理解

这一章像是：

> **租下一间空店铺，并完成开业前的基础布置。**

包括：

- 租店面 → 创建 Ubuntu 实例
- 给店铺固定门牌号 → 绑定静态 IP
- 设置门禁规则 → 配置防火墙
- 拿钥匙进店 → SSH 登录
- 检修店内设施 → 更新系统
- 给店长开工作账号 → 创建普通部署用户
- 搬入基础工具 → 安装常用软件

# 5. 部署 MySQL 8

## 5.1 本章目标

安装并配置 MySQL 8，为业务系统提供安全、规范的数据存储环境。

---

## 5.2 安装 MySQL 8

### 目标

在服务器中安装 MySQL 8 数据库服务。

### 说明

数据库用于存储：

- 用户信息
- 订单数据
- 商品信息
- 系统配置
- 业务记录

---

## 5.3 设置 root 密码

### 目标

为数据库管理员账号设置安全密码。

### 原则

- 使用强密码
- 不将 root 用于应用日常连接
- root 只用于数据库管理

---

## 5.4 创建业务数据库

### 目标

为当前项目创建独立数据库。

### 示例命名

- `project_db`
- `app_prod`
- `shop_system`

---

## 5.5 创建业务用户并授权

### 原则

- 应用程序不要使用 root 连接数据库
- 为业务系统创建独立账号
- 仅授予必要权限

### 作用

降低误操作和安全风险。

---

## 5.6 修改字符集为 utf8mb4

### 目标

统一数据库字符集，支持：

- 中文
- 英文
- 特殊符号
- Emoji

### 建议

数据库、表、连接字符集尽量统一使用 `utf8mb4`

---

## 5.7 配置远程访问策略

### 核心建议

MySQL 尽量只监听：

```text
127.0.0.1
```

### 原因

- 降低公网暴露风险
- 仅允许本机应用连接
- 更符合常见单机部署安全策略

---

## 5.8 做定时备份

### 建议方式

- 开启自动备份
- 使用 `mysqldump`
- 将备份保存到独立位置
- 定期检查备份可恢复性

---

## 5.9 运维关注点

- MySQL 尽量只监听 `127.0.0.1`
- 应用通过本机连接数据库
- 开启自动备份或定时 `mysqldump`
- 关注磁盘空间增长
- 关注数据库日志增长

---

## 5.10 本章验证方式

- MySQL 服务正常运行
- root 能正常登录
- 业务数据库已创建
- 业务用户能正常连接目标数据库
- 字符集配置正确
- 本机连接正常，外网不可直连
- 备份任务可正常执行

---

## 5.11 生活化理解

这一章像是：

> **在店里建立一个安全的档案室 / 账本中心。**

对应关系：

- 安装 MySQL → 建档案室
- 设置 root 密码 → 给总管理员上锁
- 创建业务数据库 → 划分专属档案区
- 创建业务用户 → 给普通员工发工作钥匙
- `utf8mb4` → 支持各种文字和符号的档案格式
- 只监听 `127.0.0.1` → 档案室只开内部门
- 定时备份 → 每天复印账本
- 关注空间 → 防止档案柜放满

# 6. 部署后端服务

## 6.1 本章目标

将后端 API 服务部署到服务器中，使其可以稳定运行、连接数据库、记录日志，并能在重启后自动恢复。

---

## 6.2 上传后端代码

### 常见方式

- `git clone`
- `scp` 上传
- CI/CD 发布
- Docker 镜像拉取

### 目标

将业务代码部署到服务器指定目录中。

---

## 6.3 安装依赖

### 示例

- Node.js：`npm install`
- Java：安装 JDK / 依赖构建产物
- Python：`pip install -r requirements.txt`

### 目标

安装后端服务运行所需依赖。

---

## 6.4 配置环境变量

### 常见内容

- 数据库地址
- 数据库用户名与密码
- 运行端口
- JWT 密钥
- 第三方 API Key
- 日志目录
- 运行环境标识

### 建议

- 使用 `.env`
- 或使用 systemd / pm2 环境配置
- 不将敏感配置直接写入代码仓库

---

## 6.5 连接 MySQL

### 核心内容

- host
- port
- db name
- username
- password

### 原则

应用通过本机连接 MySQL。

---

## 6.6 启动服务

### 常见监听地址

```text
127.0.0.1:8080
```

### 原则

- 后端通常只监听本机或内网端口
- 不直接对外暴露公网端口
- 由 Nginx 做统一入口转发

---

## 6.7 配置进程守护

### 目标

保证服务：

- 崩溃后自动重启
- 服务器重启后自动启动
- 可被统一查看状态和日志

---

## 6.8 常见运行方式

### Node

- 使用 `pm2`

### Java

- 使用 `systemd` 启动 jar

### Python

- 使用 `gunicorn + systemd`

---

## 6.9 关键任务清单

- 配 `.env`
- 配数据库连接
- 配日志目录
- 配 `systemd / pm2` 开机自启
- 确认后端只监听内网或本机端口，如 `127.0.0.1:8080`

---

## 6.10 本章验证方式

- 代码已成功上传
- 依赖已安装完成
- 环境变量配置正确
- 后端服务可正常启动
- 后端能正常连接 MySQL
- 守护进程状态正常
- 服务器重启后服务可自动恢复
- 本机端口可访问，公网端口未直接暴露

---

## 6.11 生活化理解

这一章像是：

> **让真正办业务的员工开始上岗，并保证他们一直在岗。**

对应关系：

- 上传代码 → 把业务手册送到店里
- 安装依赖 → 给员工配齐工作工具
- 配环境变量 → 告诉员工岗位参数
- 连接 MySQL → 给员工档案室门卡
- 启动服务 → 员工正式上班
- 配进程守护 → 安排值班与自动补岗机制
- 监听本机端口 → 员工在后台办公，不直接面向大街

---

# 7. 部署前端并配置 Nginx

## 7.1 本章目标

将前端页面部署到服务器，通过 Nginx 托管静态资源，并将 `/api` 请求转发到后端服务。

---

## 7.2 前端项目打包

### 目标

将前端源码打包为可部署的静态文件。

### 常见产物

- HTML
- CSS
- JavaScript
- 图片与字体资源

---

## 7.3 上传静态文件到服务器

### 推荐目录

```text
/var/www/project-frontend
```

### 目标

将打包产物上传到服务器指定目录中。

---

## 7.4 Nginx 托管前端页面

### 目标

由 Nginx 负责返回前端页面和静态资源。

### 路由规则

- `/` → 前端静态文件

---

## 7.5 配置 `/api` 反向代理

### 目标

将前端发出的接口请求转发到后端服务端口。

### 路由规则

- `/api/` → 代理到后端服务

---

## 7.6 处理前端路由 history 模式

### 目标

支持 Vue / React 等前端 history 路由模式。

### 原则

当用户直接访问前端内部路径时：

- 如果不是实际存在的文件
- 返回 `index.html`
- 再交由前端路由接管

---

## 7.7 开启 gzip 与缓存

### gzip

压缩文本资源，减少网络传输体积。

### 缓存

对静态资源设置缓存策略，减少重复下载并提升访问速度。

---

## 7.8 典型目录结构

```text
/var/www/project-frontend
```

Nginx 配置逻辑：

- `/` → 前端静态文件
- `/api/` → 代理到后端服务端口

这样用户只访问一个域名即可。

---

## 7.9 本章验证方式

- 前端已完成打包
- 静态文件已上传到服务器
- Nginx 可正常返回首页
- `/api` 请求可正常代理到后端
- history 路由刷新不报 404
- gzip 与缓存响应头生效

---

## 7.10 生活化理解

这一章像是：

> **把店面装修给顾客看，并让前台统一接待顾客。**

对应关系：

- 前端打包 → 把设计稿变成成品门面
- 上传静态文件 → 把装修材料运到店里
- Nginx 托管页面 → 前台直接展示宣传资料
- `/api` 反向代理 → 前台把业务请求转后台
- history 路由 → 顾客从任意入口都能回到主大厅
- gzip / 缓存 → 资料传得更快、重复访问更省事

---

# 8. 域名、HTTPS 与运维

## 8.1 本章目标

完成域名上线、HTTPS 配置以及项目长期运维能力建设。

---

## 8.2 域名解析到 Lightsail 静态 IP

### 目标

将域名指向服务器公网静态 IP。

### 常见方式

添加 DNS A 记录：

- 主机记录：`@`
- 值：服务器静态 IP

如有子域名，再添加：

- 主机记录：`www`
- 值：服务器静态 IP

---

## 8.3 申请 SSL 证书

### 常见方案

- Let's Encrypt
- 商业证书
- 云平台托管证书

### 推荐

普通项目优先考虑：

- **Let's Encrypt + certbot**

---

## 8.4 配置 HTTPS

### 目标

让网站支持 `https://` 访问，并启用加密通信。

### 建议

- 配置 443 端口
- 配置证书和私钥
- 配置 HTTP 跳转 HTTPS
