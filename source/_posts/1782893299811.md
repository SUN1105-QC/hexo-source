# 个人开发者前后端分离项目部署方案（2026 高性价比推荐）

## 一、总体方案

对于个人开发、中小型项目（约 1～1000 DAU）来说，推荐采用以下部署方案：

> **1 台 AWS Lightsail（Ubuntu） + Docker Compose**

将所有服务部署在同一台服务器：

- Vue（前端）
- Spring Boot（后端）
- MySQL（数据库）
- Nginx（反向代理）

该方案具有以下优势：

- ✅ 成本低
- ✅ 部署简单
- ✅ 运维容易
- ✅ 学习价值高
- ✅ 后期容易扩展
- ✅ 可同时部署多个项目

# 四、为什么推荐 Docker Compose？

## 一键启动

```bash
docker compose up -d
```

## 一键停止

```bash
docker compose down
```

## 一个文件管理所有服务

```text
docker-compose.yml
```

例如：

```yaml
services:
  nginx:
  frontend:
  backend:
  mysql:
```

优点：

- 服务统一管理
- 易于维护
- 配置清晰
- 后期扩展方便

---

# 五、推荐服务器配置

## 推荐配置（个人开发）

| 配置 | 推荐 |
|------|------|
| CPU | 2 vCPU |
| 内存 | 2 GB |
| SSD | 60 GB |

可稳定运行：

- Spring Boot
- Vue
- MySQL
- Nginx
- Redis（未来扩展）

---

## 最低配置

如果预算有限：

| 配置 | 最低要求 |
|------|-----------|
| CPU | 1 vCPU |
| 内存 | 1 GB |

虽然可以运行，但 Spring Boot + MySQL 会比较吃内存。

**推荐至少使用 2 GB 内存。**

# 七、Docker 网络

Docker Compose 会自动创建内部网络，例如：

```text
project-network
```

服务之间通过容器名称通信：

```text
Frontend
      │
      ▼
Nginx
      │
      ▼
Backend
      │
      ▼
MySQL
```

访问方式：

```text
http://backend:8080

mysql:3306
```

无需暴露数据库公网端口。

# 十、数据库

推荐：

> MySQL 8

数据持久化：

```yaml
volumes:
  mysql-data:
```

即使执行：

```bash
docker compose down
```

数据库数据也不会丢失。

---

# 十一、日志管理

推荐将日志映射到宿主机：

```text
logs/
```

例如：

```text
logs/

├── backend.log
├── error.log
└── access.log
```

方便排查问题。

---

# 十二、Redis

第一版项目：

**不推荐。**

很多项目一开始就引入：

- Redis
- RabbitMQ
- Kafka
- Elasticsearch

实际上，对于个人项目来说：

> MySQL 已经足够。

等访问量增加后，再逐步引入 Redis。

---

# 十三、MinIO（文件存储）

如果项目涉及：

- 图片上传
- 用户头像
- 文件上传

推荐增加：

> MinIO

Docker Compose：

```text
compose

├── nginx
├── frontend
├── backend
├── mysql
└── minio
```

方便管理文件。

# 十四、监控

后期可增加：

- Prometheus
- Grafana

- CPU
- 内存

- MySQL

第一版无需引入。

# 十五、CI/CD 自动部署

推荐：

GitHub

↓

GitHub Actions

↓

SSH 登录服务器

↓

docker compose pull

↓

docker compose up -d

这样：

每次 Push 到 GitHub 即可自动部署。
